# Max Бот-Секретарь 🤖

Автоматизированный бот для приема заявок на консультации через Max мессенджер с поддержкой GDPR и согласий на обработку персональных данных.

## Функции

- ✅ Полный цикл приема заявок через Max API
- 📋 Двух-уровневая система согласий (GDPR compliant)
- 👤 Поддержка физических и юридических лиц
- 📂 Категоризация юридических вопросов
- 💾 Сохранение заявок в SQLite БД
- 📱 Отправка уведомлений администратору
- ✔️ Валидация номеров телефонов
- 🔐 Логирование всех операций
- 🌐 Webhook API для приема сообщений

## Требования

- Python 3.11+
- Max Bot API Token
- Max Admin User ID для получения уведомлений

## Установка

### Локально

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Создайте `.env` файл:
```bash
cp .env.example .env
```

3. Установите переменные окружения:
```
MAX_BOT_TOKEN=ваш_токен_бота
MAX_ADMIN_USER_ID=240134783
```

4. Запустите бота:
```bash
python main.py
```

Бот запустится на `http://localhost:8080/webhook`

### Docker

```bash
docker build -t max-secretary-bot .
docker run -e MAX_BOT_TOKEN=your_token -e MAX_ADMIN_USER_ID=240134783 max-secretary-bot
```

### Railway

1. Создайте проект на railway.app
2. Подключите Git репозиторий
3. Установите переменные:
   - `MAX_BOT_TOKEN`
   - `MAX_ADMIN_USER_ID`
4. Развертывание произойдет автоматически

## Структура файлов

- `main.py` - точка входа
- `max_bot.py` - основная логика бота и webhook сервер
- `config.py` - конфигурация из переменных окружения
- `database.py` - SQLite операции
- `test_bot.py` - модульные тесты

## Архитектура

Бот использует HTTP API Max напрямую вместо сторонних библиотек:
- Webhook сервер на aiohttp принимает сообщения на `/webhook`
- Состояния пользователя хранятся в памяти (`user_states`, `user_data`)
- Кнопки передаются через `inline_keyboard` в JSON payload
- Все заявки сохраняются в SQLite базе

## Поток пользователя

1. /start → показать кнопку "Записаться"
2. Согласие на обработку ПД (двухшаговое)
3. Выбор типа клиента (физ./юр. лицо)
4. Выбор категории вопроса
5. Ввод имени
6. Ввод телефона (с валидацией)
7. Опциональное описание ситуации
8. Заявка сохраняется и отправляется администратору
